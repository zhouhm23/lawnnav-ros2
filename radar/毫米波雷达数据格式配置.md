#  毫米波雷达数据格式配置

EVB1122 评估套件支持 DSRAW、Range FFT 以及 Doppler FFT 等数据格式，各个数据帧格式分别参考表 5-1、表 5-2，以及表 5-3。MCU 会对各个数据帧进行封包后输出，封包的格式参考表 5-4。

## 表 5-1 DS RAW 数据格式

| Header[Dword 0] | 'b1010 1010'                | [23]                            | 'b 010' | RAW_chirp_cnt (bit 7,0,8)[19:11] | RAW_DATA_cnt[10:0] |
| --------------- | --------------------------- | ------------------------------- | ------- | -------------------------------- | ------------------ |
| Data[Dword 1]   | DS RAW real data 0[31:16]   | DS RAW Imaginary data 0[15:0]   |         |                                  |                    |
| Data[Dword 2]   | DS RAW real data 1[31:16]   | DS RAW Imaginary data 1[15:0]   |         |                                  |                    |
| Data[Dword ...] | DS RAW real data ...[31:16] | DS RAW Imaginary data...[15:0]  |         |                                  |                    |
| Data[Dword m]   | DS RAW real data m-1[31:16] | DS RAW Imaginary data m-1[15:0] |         |                                  |                    |
| Tail[Dword m+1] | Check_sum[31:16]            | RAW_FRAME_INDEX  [15:12]        | 'b 00   | CFG_MSG [9:8]                    | 'b 0101 0101       |

**说明：**

1. [23]：对于 RX1 该数值为 0，对于 RX2 该数值为 1。
2. RAW_chirp_cnt(bit 7,0,8)[19:11]：一帧数据中 chirp 的序号，序号从 0 开始。第 11 位数据为 MSB，19：12 位数据为 MSB-1 至 LSB。
3. RAW_DATA_cnt[10:0]：m 的数值，即此 chirp 中的 DS RAW 数据个数。
4. Check_sum[31:16]：本帧数据的所有数据之和，也等于低 16 位的数据之和。
5. RAW_FRAME_INDEX[15:12]：帧数序号，序号从 0 开始。
6. CFG_MSG[9:8]：用户定义的数据。

## 表 5-2 Range FFT 数据格式

| Header[Dword 0] | 'b1010 1010'             | [23]                         | 'b 011' | FFT_chirp_cnt (bit 7,0,8)[19:11] | CFG_FFT_TX_MAX[10:0] |
| --------------- | ------------------------ | ---------------------------- | ------- | -------------------------------- | -------------------- |
| Data[Dword 1]   | FFT real data 0[31:16]   | FFT Imaginary data 0[15:0]   |         |                                  |                      |
| Data[Dword 2]   | FFT real data 1[31:16]   | FFT Imaginary data 1[15:0]   |         |                                  |                      |
| Data[Dword ...] | FFT real data ...[31:16] | FFT Imaginary data...[15:0]  |         |                                  |                      |
| Data[Dword m]   | FFT real data m-1[31:16] | FFT Imaginary data m-1[15:0] |         |                                  |                      |
| Tail[Dword m+1] | Check_sum[31:16]         | FFT_FRAME_INDEX  [15:12]     | 'b 00   | CFG_MSG [9:8]                    | 'b 0101 0101         |


1. [23]：对于 RX1 该数值为 0，对于 RX2 该数值为 1。
2. FFT_chirp_index1(bit 7,0,8)[19:11]：一帧数据中 chirp 的序号，序号从 0 开始。第 11 位数据为 MSB，19：12 位数据为 MSB-1 至 LSB。
3. CFG_FFT_TX_MAX[10:0]：m 的数值，即此 chirp 中的 FFT 输出点数的个数。
4. Check_sum[31:16]：本帧数据的所有数据之和，也等于低 16 位的数据之和。
5. FFT_FRAME_INDEX[15:12]：帧数序号，序号从 0 开始。
6. CFG_MSG[9:8]：用户定义的数据。

## 表 5-3 Doppler FFT 数据格式

| Header[Dword 0] | 'b1010 1010'              | [23]                          | 'b 100' | DFFT_chirp_cnt (bit 7,0,8)[19:11] | DW_LEN[10:0] |
| --------------- | ------------------------- | ----------------------------- | ------- | --------------------------------- | ------------ |
| Data[Dword 1]   | DFFT real data 0[31:16]   | DFFT Imaginary data 0[15:0]   |         |                                   |              |
| Data[Dword 2]   | DFFT real data 1[31:16]   | DFFT Imaginary data 1[15:0]   |         |                                   |              |
| Data[Dword ...] | DFFT real data ...[31:16] | DFFT Imaginary data...[15:0]  |         |                                   |              |
| Data[Dword m]   | DFFT real data m-1[31:16] | DFFT Imaginary data m-1[15:0] |         |                                   |              |
| Tail[Dword m+1] | Check_sum[31:16]          | FFT_FRAME_INDEX  [15:12]      | 'b 00   | CFG_MSG [9:8]                     | 'b 0101 0101 |


**说明：**
1. [23]：对于 RX1 该数值为 0，对于 RX2 该数值为 1。
2. DFFT_chirp_index1(bit 7,0,8)[19:11]：一帧数据中 chirp 的序号，序号从 0 开始。第 11 位数据为 MSB，19：12 位数据为 MSB-1 至 LSB。
3. DW_LEN[10:0]：Doppler FFT 数据长度，序号从 0 开始。
4. Check_sum[31:16]：本帧数据的所有数据之和，也等于低 16 位的数据之和。
5. FFT_FRAME_INDEX[15:12]：帧数序号，序号从 0 开始。
6. CFG_MSG[9:8]：用户定义的数据。

## 表 5-4 MCU 封包格式

| 帧头                   | 帧内数据长度 | 类型 | 通道号 | 数据   | 帧尾                   |
| ---------------------- | ------------ | ---- | ------ | ------ | ---------------------- |
| 0x49, 0x43, 0x4C, 0x48 | 2 字节       | 0x00 | 1 字节 | 数据段 | 0x49, 0x43, 0x4C, 0x54 |

## **二、与PDF手册的关键差异**

| **项目** | **手册描述** | **代码实现** | **说明** |
|----------|--------------|--------------|----------|
| **字节序** | 未明确说明（表格显示LSB在左） | **大端序** (`>I`, `>h`) | **关键修正**：实际测试验证为大端序传输 |
| **MCU帧长度字段** | 2字节（小端序隐含） | `struct.unpack('<H', ...)` | **混合字节序**：MCU长度用小端序，雷达数据用大端序 |
| **同步策略** | 无说明 | 逐字节滑窗 + 100次错误重置 | 增强健壮性，防止死锁 |
| **校验和范围** | "所有数据之和" | 包含Header+Data+Tail（除checksum自身） | 需实验确认是否包含帧尾最后4字节 |
| **数据点数量** | `m`由`RAW_DATA_cnt`等字段定义 | 从Header[10:0]位提取 | 实际为`m+1`（含1个尾部dword） |
| **帧索引位置** | Tail[dword m+1]的[15:12]位 | 倒数第2个dword的[15:12]位 | 与手册一致 |

## **六、核心算法伪代码**

### **复数数据提取**
```python
def extract_complex(radar_seg: bytes, n_points: int) -> List[complex]:
    """从雷达帧提取复数数组"""
    complex_data = []
    for i in range(n_points):
        idx = 4 + i * 4  # 跳过4字节Header
        real = struct.unpack('>h', radar_seg[idx:idx+2])[0]  # 大端序有符号16位
        imag = struct.unpack('>h', radar_seg[idx+2:idx+4])[0]
        complex_data.append(complex(real, imag))
    return complex_data
```

### **校验和手动验证**
```python
def manual_checksum_check(radar_seg: bytes):
    """手动验证校验和算法"""
    # 1. 提取尾部校验和
    tail_dword = struct.unpack('>I', radar_seg[-8:-4])[0]
    expected = (tail_dword >> 16) & 0xFFFF
    
    # 2. 计算累加和（除最后4字节）
    calc_sum = 0
    for i in range(0, len(radar_seg)-4, 4):
        dword = struct.unpack('>I', radar_seg[i:i+4])[0]
        calc_sum += (dword & 0xFFFF) + ((dword >> 16) & 0xFFFF)
    
    # 3. 比较低16位
    return (calc_sum & 0xFFFF) == expected
```
